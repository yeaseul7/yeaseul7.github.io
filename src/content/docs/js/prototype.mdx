### Prototype 기반 OOP

> Javascript는 Prototype으로 OOP 특성인 상속성을 보여준다.

생성자 함수를 사용하여 OOP 특성인 상속성을 구현하면 함수 같은 경우 인스턴스마다 속성이 중복으로 생기기 때문에 메모리 측면에서 비효율적일 수 있다.<br/>
이때 prototype으로 한곳에 함수를 저장하고 prototype 접근을 통해 효율적으로 사용할 수 있다.<br/>
이것을 프로토타입 체인(Prototype Chain)을 통한 메서드 공유라고 한다.

```javascript
// 생성자 함수를 사용하는 경우
function Circle(radius) {
  this.radius = radius;
  this.getArea = function () {
    return Math.PI * this.radius ** 2;
  };
}

const circle1 = new Circle(1);
const circle2 = new Circle(2);

//getArea가 중복으로 생성됨 🥲

// Prototype을 사용하는 경우

function Circle(radius) {
  this.radius = radius;
}

Circle.prototype.getArea = function () {
  return Math.PI * this.radius ** 2;
};

const circle1 = new Circle(1);
const circle2 = new Circle(2);

//Circle.prototype으로부터 getArea를 상속받는다.
```

모든 객체는 [[Prototype]]이 존재하고 그 값은 null일 수 있다.<br/>
객체 생성 방식에 따라 어떤 값이 들어갈지 정해진다.<br/>
또한 그 값은 `__proto__`로 간접 접근 가능하다.

### [[Prototype]] & prototype

**`[[Prototype]]`**

- `__proto__` 접근자나 `Object.getPrototypeOf()`로 접근 가능하다.
- 상위에 있는 프로토타입 객체를 의미한다.
- 함수의 경우 -> `Function.prototype` 를 가리킨다.
- 인스턴스의 경우 -> 생성자 함수의 `prototype` 속성이 가리키는 객체를 의미한다.

<pre> 과거에는 __proto__를 주로 사용하였지만 최근에는 Object.getPrototypeOf()와 Object.setPrototypeOf()를 사용한다. <br/>
__proto__는 접근자 프로퍼티 이기때문에 해당 속성에 접근하면 그 객체의 부모를 강제로 바꿀 수 있다. -> Prototype Pollution <br/>
또한 Hidden Class 기법에 방해의 주된 요인이다 </pre>

**`prototype` 프로퍼티**

- 함수 객체만 가지고 있는 일반 프로퍼티
- 이 함수를 이용해서 new 연산자로 생성한 인스턴스의 부모로 사용될 객체를 가리킨다.
  > 여기서 말하는 부모는 [[Prototype]]을 의미한다.<br/>
  > ⚠️ 화살표 함수는 non-constructor이기 때문에 prototype 프로퍼티가 없다.
  > 화살표 함수는 생성자가 될 수 없어서 인스턴스가 없고 prototype 프로퍼티가 존재하지 않는다.<br/>

이걸 만든 이유는 함수를 100개 만들때 함수 100개가 메모리에 따로 생성되는 비효율을 막고자 이다.<br/>
이걸 만들면 공유가 가능하기 때문에 메모리 효율이 좋아진다.<br/>

```javascript
function Dog(name) {
  this.name = "early";
}

const puppy = new Dog("access");

console.log(Dog.prototype); // Dog는 함수이므로 prototype 프로퍼티 존재
console.log(puppy.prototype); // undefined

console.log(puppy.__proto__ === Dog.prototype); // true
console.log(Dog.__proto__); // Function.prototype
console.log(puppy.__proto__); // Dog.prototype
```

### `__proto__`

> `__proto__`는 접근자 프로퍼티이다.

`__proto__`는 getter/setter를 통해 [[Prototype]]에 접근할 수 있게 해준다.

```javascript
const obj = {};
const parent = { x: 1 };

// setter 함수인 set __proto__가 호출되어 obj 객체의 프로토타입을 교체
obj.__proto__ = parent;
```

#### `__proto__`를 사용하는 이유

`__proto__`를 사용하지 않으면 비정상적인 프로토타입 체인이 만들어질 수 있다.

js 엔진이 제공하는 **proto**의 setter는 순환참조를 감시한다.

```javascript
A.__proto__ = B;
```

위처럼 쓰면 setter가 호출된다.<br/>
그러면 stter가 B가 이미 조상인지 (A의 프로토타입 체인에 존재하는지) 확인한다.<br/>
만약 이미 존재한다면 `TypeError: Cyclic __proto__ value`를 발생시켜서 막는다.

```javascript
const parent = {};
const child = {};
// child의 프로토타입을 parent로 설정
child.__proto__ = parent;
// parent의 프로토타입을 child로 설정
parent.__proto__ = child; // TypeError: Cyclic __proto__ value
```

**프로토타입 체인은 Linked List 구조이기 때문에 순환 참조가 발생하면 안된다.**

접근자 프로퍼티를 사용하지 않고 직접 변경하는 방법은 없지만 그렇게 하면 체인을 탐색하는 과정에서 무한루프가 발생할 수 있다.

### Prototype Chain

객체에서 프로퍼티나 메소드에 접근하려고 할 때 자신에게 해당 속성이 없으면 [[Prototype]]이 가리키는 링크를 따라 탐색한다.

1. 해당 객체 자신의 프로퍼티를 참조한다.
2. 없다면 -> [[Prototype]] 내부 슬롯의 참조를 따라 탐색한다.

계속 올라가다가 null을 프로토타입으로 가지는 Object를 만나면 그때 프로퍼티 참조를 멈춘다.
이러한 과정을 프로토타입 체인이라고 한다.

```javascript
const early = {
  type: "Dog",
};

console.log(early.hasOwnProperty("type")); // -> true

//early라는 객체는 hasOwnProperty라는 메소드를 가지고 있지 않는데 정상적으로 출력되었다.
//이는 early라는 객체의 [[Prototype]]이 가리키는 링크인 Object.prototype의 hasOwnProperty를 호출하였기 때문이다.
//Object.prototype.hasOwnProperty.call(early, "type")
```

javascript의 Reference Type은 모두 위와 같은 방식으로 동작한다. (ex. Array, Set, Object, Function)
Reference Type은 위의 프로토타입 체인을 통해 내장 메서드에 접근한다. (ex. Array.prototype.map())

```javascript
function Person(name) {
  this.name = name;
}
// 프로토타입 메서드
Person.prototype.sayHello = function () {
  console.log(`Hi! My name is ${this.name}`);
};
const me = new Person("Lee");
// hasOwnProperty는 Object.prototype의 메서드다.
console.log(me.hasOwnProperty("name")); // true
```

### 객체 리터럴 방식

JavaScript 엔진은 객체 리터럴로 객체를 생성할 때 내부적으로 Object.prototype을 [[Prototype]]으로 갖는 객체를 만든다. <br/>

> 객체 리터럴도 사실 엔진 내부에서는 new Object()를 부르는 것과 똑같이 프로토타입 체인을 가진다<br/>

이는 Object.create(Object.prototype)로 객체를 생성하는 방식과 유사하게 이해할 수 있다.<br/>
Object.create()는 Object 생성자 함수의 정적 메서드이며 인자로 전달받은 객체를 새 객체의 [[Prototype]]으로 설정한다.

OrdinaryObjectCreate
`const obj = {}`라고 작성하면 내부적으로 `OrdinaryObjectCreate(Object.prototype)`를 호출한다.<br/>

⚠️ 객체 리터럴은 new Object()처럼 개발자가 눈에 보이는 생성자 함수를 직접 호출하는 방식이 아니기 때문에 엔진이 판단하기에 따라 연결이 달라질 수 있는 여지가 있다

```javascript
// 1. Object 생성자 함수에 의한 객체 생성
// 인수가 전달되지 않았을 때 추상 연산 OrdinaryObjectCreate를 호출하여 빈 객체를 생성한다.
let obj = new Object();
console.log(obj); // {}

// 2. new.target이 undefined나 Object가 아닌 경우
// 인스턴스 -> Foo.prototype -> Object.prototype 순으로 프로토타입 체인이 생성된다.
class Foo extends Object {}
new Foo(); // Foo {}

// 3. 인수가 전달된 경우에는 인수를 객체로 변환한다.
// Number 객체 생성
obj = new Object(123);
console.log(obj); // Number {123}

// String 객체 생성
obj = new Object("123");
console.log(obj); // String {"123"}
```

`Object.create(null)`은 Object.prototype을 상속받지 않는 객체를 생성한다.<br/>
즉, 연결을 끊어버린다.

이걸 통해서 프로토타입 체인이 `필수 법칙`이 아니고 편의 기능과 같다고 이해할 수 있다.<br/>
언제든 Object.create(null)로 프로토타입 체인을 끊을 수 있기 때문에.

```javascript
const obj = {};
const obj2 = new Object();
// obj와 obj2는 동일한 프로토타입 체인을 가진다.
```

### 팁

Polyfill : 브라우저가 지원하지 않는 최신 기능을 `존재하지 않을 때에만` 기존의 내장 객체의 프로토타입에 추가하는 것

```javascript
if (!Array.prototype.includes) {
  Array.prototype.includes = function (value) {
    return this.indexOf(value) !== -1;
  };
}
```

Ponyfill : 내장 객체를 직접 수정하지 않고 독립적인 함수로 제공하는 방법

```javascript
function includes(array, value) {
  return array.indexOf(value) !== -1;
}

includes([1, 2, 3], 2); // true
```

네이티브 프로토타입을 확장하면 캡슐화를 망친다.
(Monkey Patching: 기존의 객체나 함수의 동작을 런타임에 직접 수정하거나 덮어쓰는 것)
