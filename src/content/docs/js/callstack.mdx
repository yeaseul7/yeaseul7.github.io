콜 스택은 JavaScript 엔진이 현재 어떤 함수가 실행 중인지 함수 실행이 끝난 뒤 어디로 돌아가야 하는지를 관리하기 위해 사용하는 스택 구조이다.

JavaScript에서 함수가 호출되면 해당 함수의 실행 컨텍스트가 콜 스택에 쌓인다.<br/>
함수 실행이 끝나면 해당 실행 컨텍스트는 콜 스택에서 제거되고 이전 실행 컨텍스트로 돌아간다.

콜 스택은 LIFO, 즉 Last In First Out 구조를 가진다.<br/>
가장 마지막에 호출된 함수가 가장 먼저 실행을 끝내고 자신을 호출했던 위치로 돌아가야 하기 때문이다.

그 이유를 설명해보면

```javascript
function drawSquare() {
  drawLine();
  drawLine(); // 실행 후
  drawLine(); // 여기로 돌아와야 함
  drawLine();
}
```

위 코드에서 drawSquare 함수가 실행되는 도중 drawLine 함수가 호출된다.<br/>
drawLine 함수가 실행되는 동안에는 drawSquare의 실행이 잠시 멈춘다.<br/>
drawLine 함수가 끝나면 다시 drawSquare 함수의 다음 줄로 돌아와야 한다.

`함수 호출은 가장 안쪽에서 호출된 함수부터 먼저 끝나야 한다.`<br/>
이 흐름이 스택의 LIFO 구조와 잘 맞기 때문에 JavaScript 엔진은 콜 스택을 사용한다.

### 콜 스택과 실행 컨텍스트

JavaScript에서 함수가 호출되면 실행 컨텍스트가 생성된다.<br/>
실행 컨텍스트는 현재 실행 중인 코드가 어떤 환경에서 실행되는지를 나타내는 정보이다.

실행 컨텍스트에는 대략 다음과 같은 정보가 포함된다.

- 현재 실행 중인 코드의 위치
- 변수와 함수 선언에 대한 정보
- this 바인딩
- 외부 렉시컬 환경에 대한 참조

함수가 호출되면 실행 컨텍스트가 콜 스택에 쌓이고, 함수 실행이 끝나면 콜 스택에서 제거된다.

### 콜 스택과 메모리 관리

콜 스택은 함수 호출과 종료 순서가 명확하기 때문에 메모리를 빠르게 관리할 수 있다.
함수가 호출되면 필요한 정보가 스택에 쌓이고 함수가 끝나면 위에서부터 제거된다.

이 과정은 push와 pop 방식으로 이루어지기 때문에 비교적 단순하고 빠르다.

하지만 힙은 객체나 배열처럼 수명이 일정하지 않은 데이터를 저장하는 공간이다.
어떤 객체는 함수가 끝난 뒤에도 다른 곳에서 참조될 수 있기 때문에, 스택처럼 단순히 위에서부터 제거할 수 없다.

```javascript
function createUser() {
  const user = {
    name: "철수",
  };

  return user;
}

const result = createUser();
```

위 코드에서 createUser 함수의 실행 컨텍스트는 함수가 끝나면 콜 스택에서 제거된다.<br/>
하지만 user 객체는 result가 참조하고 있기 때문에 바로 사라지지 않는다.

함수 실행 자체는 콜 스택에서 관리되지만 객체처럼 함수 밖에서도 사용될 수 있는 데이터는 힙에 남을 수 있다.

### 재귀와 스택 오버플로

재귀 함수는 함수 내부에서 자기 자신을 다시 호출하는 함수이다.

```javascript
function count(n) {
  if (n === 0) return;

  console.log(n);
  count(n - 1);
}
```

재귀 함수가 호출될 때마다 새로운 실행 컨텍스트가 콜 스택에 쌓인다.
재귀의 깊이가 너무 깊어지면 콜 스택의 한계를 초과할 수 있다.

이때 발생하는 오류가 스택 오버플로이다.

```javascript
function infinite() {
  infinite();
}

infinite();
```

위 코드는 종료 조건 없이 자기 자신을 계속 호출한다.
그 결과 콜 스택에 실행 컨텍스트가 계속 쌓이고, 결국 콜 스택의 한계를 넘게 된다.

그래서 깊이가 예측되지 않는 재귀는 반복문이나 명시적인 자료구조로 바꾸는 것이 더 안전할 수 있다.

```javascript
// 재귀 대신 반복문 사용
function count(n) {
  while (n > 0) {
    console.log(n);
    n--;
  }
}
```

### 함수 안에 함수가 있을 때 변수는 어떻게 찾을까?

JavaScript는 변수를 찾을 때 현재 스코프에서 시작해 바깥 스코프로 이동하며 변수를 탐색한다.<br/>
이 구조를 스코프 체인이라고 한다.

```javascript
function outer() {
  const name = "철수";

  return function inner() {
    console.log(name);
  };
}

const innerFunc = outer();
innerFunc();
```

#### 중첩 함수의 스코프 탐색 (스코프 체인)

1. inner 스코프에서 name 찾기
2. 없으면 outer 스코프에서 name 찾기
3. 없으면 전역 스코프에서 name 찾기

함수 안에 함수가 선언되면 안쪽 함수는 자신이 선언된 위치의 외부 렉시컬 환경을 참조할 수 있다. <br/>
JavaScript는 변수를 찾을 때 현재 스코프에서 바깥 스코프로 스코프 체인을 따라 탐색한다.<br/>
이 구조 덕분에 내부 함수는 외부 함수가 종료된 뒤에도 필요한 변수를 기억할 수 있다. -> Closure

#### 스택 프레임이 존재하는 이유

함수가 동작할때마다 별도의 스택 프레임이 없다면, 변수가 뒤섞여서 js에서 스코프 체인이 작동할 수 없다.

```
Error: 문제 발생
at third
at second
at first
```

최근 JavaScript 엔진은 비동기 흐름을 더 잘 추적하기 위해 async stack trace를 제공하기도 한다.
