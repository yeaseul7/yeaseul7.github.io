## 타입 호환성

TypeScript에서는 타입 이름이 달라도, 필요한 속성과 메서드를 가지고 있으면 같은 타입처럼 사용할 수 있다.

```typescript
interface Food {
  name: string;
  caloriePer100g: number;
  weightInGram: number;
}

function calcCalorie(food: Food) {
  return (food.caloriePer100g * food.weightInGram) / 100;
}

// 1. Food 타입과 정확히 일치
const apple: Food = {
  name: "사과",
  caloriePer100g: 52,
  weightInGram: 150,
};

calcCalorie(apple); // 가능

// 2. Food 타입과 전혀 다른 구조
interface Computer {
  brand: string;
  cpu: string;
  ramGB: number;
}

const myComputer: Computer = {
  brand: "LG",
  cpu: "Intel i7",
  ramGB: 16,
};

calcCalorie(myComputer); // 에러

// 3. Food와 일부 속성만 일치
interface Burger {
  name: string;
  caloriePer100g: number;
}

const burger: Burger = {
  name: "치즈버거",
  caloriePer100g: 295,
};

calcCalorie(burger); // 에러

// 4. Food라고 선언하지 않았지만 구조가 같음
const burgerWithWeight = {
  name: "치즈버거",
  caloriePer100g: 295,
  weightInGram: 220,
};

calcCalorie(burgerWithWeight); // 가능

// 5. Food보다 속성이 더 많음
const chickenBurger = {
  name: "치킨버거",
  caloriePer100g: 260,
  weightInGram: 240,
  protein: 28,
  price: 6500,
};

calcCalorie(chickenBurger); // 가능
```

만약 버거의 타입이 Food와 정확하게 일치한다면 실제 정상처럼 동작할 수 있는 코드가 된다. <br/>
그렇다면 타입 시스템은 개발자의 의도에 맞게 유연하게 대응 하는것이 좋을수도 있다.
이게 바로 TypeScript의 타입 호환성이다. <br/>

하지만 유연성만을 생각 할 순 없다. <br/>
따라서 어떤 경우에 호환이 되고 안되는지를 규칙이 필요하다. <br/>
그 규칙을 만드는 방법은 `명목적 서브타이핑(Nominal Subtyping)`과 `구조적 타이핑(Structural Typing)`이 존재한다.

### 1. 명목적 서브타이핑(Nominal Subtyping)

<pre>
  “네가 진짜로 그 타입이라고 선언됐어?” <br />
  “그 타입을 상속하거나 구현한다고 명시했어?”
</pre>

```typescript
interface Food {
  name: string;
  caloriePer100g: number;
  weightInGram: number;
}

interface Burger {
  name: string;
  caloriePer100g: number;
  weightInGram: number;
}

const burger: Burger = {
  name: "치즈버거",
  caloriePer100g: 295,
  weightInGram: 220,
};

const food: Food = burger; // 가능
```

이렇게 하면 타입 오류를 방지할 뿐더러 명확한 의도를 반영할 수 있다.

### 2. 구조적 타이핑(Structural Typing)

구조적 타이핑은 값이 가진 구조를 기준으로 타입 호환성을 판단하는 방식이다.<br/>
이름이나 명시적 선언이 필요 없다.

```typescript
interface Food {
  name: string;
  caloriePer100g: number;
  weightInGram: number;
}

function calcCalorie(food: Food) {
  return (food.caloriePer100g * food.weightInGram) / 100;
}

interface Burger {
  name: string;
  caloriePer100g: number;
  weightInGram: number;
  hasCheese: boolean;
}

const burger: Burger = {
  name: "치즈버거",
  caloriePer100g: 295,
  weightInGram: 220,
  hasCheese: true,
};

calcCalorie(burger); // 가능
```

### 타입 호환 테스트

```typescript
const burger = {
  protein: 29,
  carbohydrates: 48,
  fat: 13,
  burgerBrand: "버거킹",
};

const calorie = calculateCalorie(burger);
/** 타입검사결과 : 오류없음 (OK) */

const calorie = calculateCalorie({
  protein: 29,
  carbohydrates: 48,
  fat: 13,
  burgerBrand: "버거킹",
});
/** 타임검사결과 : 오류 (NOT OK)*/
```
