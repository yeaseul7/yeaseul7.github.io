## JavaScript가 콜 스택을 사용하는 이유

콜 스택은 JavaScript 엔진이 현재 어떤 함수가 실행 중인지와 함수 실행이 끝난 뒤 어디로 돌아가야 하는지를 관리하기 위해 사용하는 스택 구조다.
JavaScript에서 함수가 호출되면 해당 함수의 실행 컨텍스트가 콜 스택에 쌓인다. 함수 실행이 끝나면 해당 실행 컨텍스트는 콜 스택에서 제거되고 이전 실행 컨텍스트로 돌아간다.
콜 스택은 LIFO(Last In, First Out) 구조를 가진다. 가장 마지막에 호출된 함수가 가장 먼저 실행을 끝내고 자신을 호출했던 위치로 돌아가야 하기 때문이다.

```javascript
function drawSquare() {
  drawLine();
  drawLine(); // 실행 후
  drawLine(); // 여기로 돌아와야 함
  drawLine();
}
```

### 콜 스택과 메모리 관리

콜 스택은 함수 호출과 종료 순서가 정해져 있기 때문에 메모리를 빠르게 관리할 수 있다.
함수의 호출과 종료에 의한 과정은 스택의 push/pop 방식으로 이루어지기 때문에 비교적 단순하고 빠르다.

반면 힙은 객체처럼 수명이 일정하지 않은 데이터를 저장하는 데 사용된다. 힙 메모리는 필요한 공간을 찾고 참조 관계를 추적하고 더 이상 사용하지 않는 데이터를 가비지 컬렉션으로 정리해야 하므로 스택보다 관리 비용이 크다.

### 매개변수 전달과 스택

근본적으로 함수 호출 시 인자는 보통 CPU 레지스터를 통해 먼저 전달된다. 하지만 레지스터는 개수가 제한되어 있기 때문에 레지스터가 부족하거나 호출 규약상 필요한 경우 일부 인자는 스택을 통해 전달될 수 있다.

<pre>
// 호출 규약(Calling Convention)
일부 인자 → 레지스터
나머지 인자 → 스택
</pre>

이말은 곧, 스택은 함수 호출 과정에서 복귀 주소, 일부 인자, 지역 데이터, 임시 값 등을 저장하는 보조 공간으로도 사용될 수 있다.
JavaScript 개발자가 직접 레지스터나 스택을 제어하지 않는다.

### 함수 안에 함수가 있을 때 변수는 어떻게 찾을까?

```JavaScript
function outer() {
  const name = "철수";

  return function inner() {
    console.log(name);
  };
}

const innerFunc = outer();
innerFunc();
```

#### 중첩 함수의 스코프 탐색 (스코프 체인)
1. inner 스코프에서 name 찾기
2. 없으면 outer 스코프에서 name 찾기
3. 없으면 전역 스코프에서 name 찾기

함수 안에 함수가 선언되면 안쪽 함수는 자신이 선언된 위치의 외부 렉시컬 환경을 참조할 수 있다. <br/>
JavaScript는 변수를 찾을 때 현재 스코프에서 바깥 스코프로 스코프 체인을 따라 탐색한다.<br/>
이 구조 덕분에 내부 함수는 외부 함수가 종료된 뒤에도 필요한 변수를 기억할 수 있다. -> Closure

#### 스택 프레임이 존재하는 이유
함수가 동작할때마다 별도의 스택 프레임이 없다면, 변수가 뒤섞여서 js에서 스코프 체인이 작동할 수 없다. 

#### 스택 unwinding
try...catch나 비동기 호출에서 Promise가 reject 되었을 때 '스택 트레이스(Stack Trace)'가 찍힌다.
이것이 언와인딩 과정에서 오류 지점까지의 스택 프레임들을 역추적해서 보여주는 것이다.
