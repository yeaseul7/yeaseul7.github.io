#### 데이터 기반의 성능 튜닝 및 웹 아키텍처 개편 (초기 로딩 10s ➔ 3s 단축)
* **Problem**: 각각의 화면이 개별 HTML 페이지로 파편화되어 있는 레거시 구조였습니다. 이로 인해 새로운 페이지에 진입할 때마다 무거운 공통 스크립트(Vue, CDN)와 외부 라이브러리가 중복으로 다운로드되었고 불필요한 리소스 로딩 지연이 겹치면서 초기 화면이 뜨는 데 10초 이상 소요되었습니다.

* **Action**: Chrome DevTools와 Lighthouse를 활용해 병목 구간을 분석했습니다. 리소스 중복 로드 문제를 근본적으로 해결하기 위해 화면 구조를 하나의 앱으로 묶어 재사용성을 높인 React 컴포넌트 중심으로 개편했습니다. 이와 동시에 정적 리소스를 효율적으로 서빙하기 위해 Azure CDN 환경으로 마이그레이션하여 로딩 우선순위를 최적화했습니다.

* **Result**: 주요 화면 초기 렌더링 속도를 **70% 이상 대폭 개선(10초 ➔ 3초)** 하였습니다.

#### 인프라 연동 및 전사 업무 자동화

* **Problem**: 기존 휴가 신청 및 결재가 시스템 없이 전적으로 수기 확인과 개별 연락에 의존하고 있어서 불필요한 행정 공수가 상시 발생했습니다.

* **Action**: 업무 효율 개선을 위해 직접 기획부터 참여하여 **대시보드 및 커스텀 캘린더 기반의 워크플로우 시스템을 단독 구현**했습니다. 특히 수동 확인 단계를 없애고자 **Power Automate와 Microsoft Teams Webhook을 결합**하여 결재 신청·승인·반려 등 모든 진행 상태가 사내 메신저로 실시간 트리거되는 자동화 파이프라인을 구축했습니다.

* **Result**: 반복적인 휴가 확인 및 메신저 소통 비용을 제거하고 전사 행정 업무 처리 효율을 크게 향상시켰습니다.

### 2. 포유테일 (Foryoutail) — 1인 서비스 개발 및 운영
*2025.11 ~ 2025.05*

> 사용자 경험과 클라우드 운영 비용을 모두 고려하여 아키텍처를 설계합니다.<br/>
> 파편화된 유기동물 보호소 데이터를 유저들이 쉽게 찾고 소통할 수 있도록 기획, 디자인, 개발, 배포까지 단독으로 총괄한 B2C 커뮤니티 서비스입니다. (MAU 562명 달성)

#### DB 패러다임 전환을 통한 데이터 호출 비용 최적화 (NoSQL ➔ RDBMS)

* **Problem**: 초기 구조인 Firestore에서는 게시글, 좋아요, 댓글 등의 관계형 데이터를 가져올 때마다 추가 서브 쿼리가 발생했습니다. 유저가 늘어날수록 목록 조회 할 때 쿼리 호출 횟수와 클라우드 비용 리스크가 커지는 문제가 존재했습니다.

* **Action**: 비용 계산 후 Supabase PostgreSQL로 이전해서 테이블 정규화를 진행했습니다. 마이그레이션 과정에서 데이터 중복 및 누락을 막기 위해 Upsert 방식의 데이터 매핑 구조를 설계했습니다.

* **Result**: 데이터 호출 빈도를 획기적으로 줄여 인프라 비용 부담을 크게 줄였습니다.

#### 서버리스 전환을 통한 고정 비용 제거 및 보안 고도화

* **Problem**: 서버 유지 비용 부담을 최소화해야 했습니다. 또한 Supabase RLS 정책 우회 취약점을 보완해서 클라이언트 측에서의 안전한 데이터 접근 권한을 보장해야 했습니다.

* **Action**: 배포 환경을 Cloudflare Workers로 이전하여 Edge 기반 라우팅 성능을 개선하고 고정 비용 리스크를 줄였습니다. 또한 보안 강화를 위해 서버 단에서 JWT 기반으로 권한 검증을 처리하도록 인증 흐름을 구축했습니다.

* **Result**: 고정 운영 비용을 줄여서 서비스 지속 가능성을 높이고 사용자 데이터 접근 보안을 강화했습니다.

#### Pinecone 벡터 검색 기반 AI 추천 도입

* **Problem**: 단순 키워드 검색이 아니라 유저가 입력한 텍스트와 유기동물의 세부 특성을 다차원으로 매치해 주는 기능이 필요했습니다.

* **Action**: Pinecone 벡터 검색 엔진을 연동하여 유저 입력값과 동물 데이터 간의 유사도를 계산하는 지능형 추천 기능을 프론트엔드와 유기적으로 결합했습니다.

* **Result**: 사용자의 탐색 동선을 줄이고 개인화된 정보 제공 만족도를 높였습니다.
