## DP란?

Dynamic Programing으로 하나의 문제를 여러개로 나누고 기억하여 반복되는 경우 기억을 이용하여 결과를 도출하는 알고리즘 기법이다.

#### 재귀함수와 차이

재귀함수는 문제를 작은 단위로 나누는 기법(Divide and Conquer).<br/>

예를 들어서 피보나치 수열을 구하는 경우 재귀함수는 중복해서 계산하는 경우가 많이 발생한다.<br/>
f(n) = f(n-1) + f(n-2) <br/>
이기 때문에 시간 복잡도는 O(2^n)이다.<br/>

```
              F(4)
             /    \
           F(3)    F(2)
          /   \    /   \
        F(2)  F(1) F(1)  F(0)
       /   \
     F(1)  F(0)
```

하지만 DP는 중복되는 경우 Memoization을 이용해서 효율적으로 결과를 도출한다.
따라서 중복 호출을 막는 선형시간으로 O(n)으로 줄어든다.

```
              F(4)
             /    \
           F(3)    F(2) ----> [Cache Hit!] (즉시 반환)
          /   \
        F(2)  F(1) ----> [Cache Hit!] (즉시 반환)
       /   \
     F(1)  F(0)
    (종료) (종료)
```

### DP를 사용하기 위해서

1. Overlapping Subproblems : 겹치는 부분이 있는가? <br/>
2. Optimal Substructure : 작은 문제의 답이 큰 문제의 답에 영향을 주는가? <br/>

### DP 유형

- Top-Down : 재귀함수 + Memoization <br/>
  큰 문제를 위해 작은 문제로 쪼개는 방식

```javascript
// 메모이제이션 배열
const memo = new Array(100).fill(0);

function fibTopDown(n) {
  if (n <= 1) return n;

  // 이미 계산한 적 있다면 바로 반환 (Memoization)
  if (memo[n] !== 0) return memo[n];

  // 계산한 적 없다면 재귀 호출 후 저장
  memo[n] = fibTopDown(n - 1) + fibTopDown(n - 2);
  return memo[n];
}
```

- Bottom-Up : 반복문 + Tabulation <br/>
  가장 작은 문제부터 차례대로 계산하여 큰 문제를 해결하는 방식

```javascript
function fibBottomUp(n) {
  // DP 테이블 생성
  const dp = new Array(n + 1).fill(0);
  dp[1] = 1;

  // 작은 문제부터 차례대로 해결 (Tabulation)
  for (let i = 2; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2];
  }

  return dp[n];
}
```

### DP 문제를 해결하는 로직

1. 문제의 Overlapping Subproblems를 파악
2. 결과를 바꾸는 변수가 무엇인지 파악
3. Optimal Substructure를 파악 (점화식을 구한다고 한다.)<br/>
   `점화식이란 현재의 나를 이전 단계의 나로 설명하는 식` <br/>
4. Top-Down 방식인지 Bottom-Up 방식인지 결정
5. 기저상태 파악(더이상 쪼개질 수 없는 가장 작은 문제 ex. f(1)=1, f(0)=0) <br/>

### DP 문제에서 dp[i]와 dp[i][w]의 의미

- dp[i] : i번째까지의 문제를 해결했을 때의 최적의 결과값 <br/>
- dp[i][w] : w라는 조건이 더해진 것<br/>

## 배낭문제

"제한된 무게($W$)를 가진 배낭에, 각각의 무게($w$)와 가치($v$)를 가진 물건들을 담아 담을 수 있는 가치의 총합을 최대로 만드는 문제"

- 변수 파악 :
  - $i$ : 물건의 개수
  - $w$ : 배낭의 무게

- 점화식 :
- $dp[i][w]$ : $i$번째 물건까지 고려했을 때, 배낭의 무게가 $w$일 때의 최대 가치
- $dp[i][w] = max(dp[i-1][w], dp[i-1][w-w[i]] + v[i])$ <br/>
  - $dp[i-1][w]$ : $i$번째 물건을 넣지 않는 경우
  - $dp[i-1][w-w[i]] + v[i]$ : $i$번째 물건을 넣는 경우

```javascript
//knapsack(4, [2, 3], [3, 4])

function knapsack(capacity, weights, values) {
  const n = weights.length;
  // dp[물건 개수 + 1][최대 무게 + 1] -> n개가 있다면 0번째도 고려해줘야해서 +1 하는 것
  const dp = Array.from({ length: n + 1 }, () => Array(capacity + 1).fill(0));

  // [0, 0, 0, 0], // i=0: 물건이 하나도 없을 때
  // [0, 0, 0, 0], // i=1: 1번 물건을 고려할 때
  // [0, 0, 0, 0]  // i=2: 2번 물건을 고려할 때

  for (let i = 1; i <= n; i++) {
    const currentWeight = weights[i - 1];
    const currentValue = values[i - 1];

    for (let w = 0; w <= capacity; w++) {
      if (currentWeight <= w) {
        // 넣는 경우 vs 안 넣는 경우 비교
        dp[i][w] = Math.max(
          currentValue + dp[i - 1][w - currentWeight],
          dp[i - 1][w],
        );
      } else {
        // 무게가 초과되면 이전 단계 값 그대로 가져옴
        dp[i][w] = dp[i - 1][w];
      }
    }
  }
  return dp[n][capacity];
}
```
